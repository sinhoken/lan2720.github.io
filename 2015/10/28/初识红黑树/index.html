初识红黑树

发表于 2015-10-28   |  

什么是红黑树

红黑树是表示2-3树的二叉查找树【细细品味】。从这个定义可以看出，红黑树既是一个2-3树，又是一个二叉查找树。这点很关键因为这么一来红黑树就把两个算法的优点都结合起来了：

  • 二叉查找树中简洁高效的查找方法
  • 2-3树中高效的平衡插入算法

乍一看红黑树很神奇，听起来和《红与黑》一样挺惊悚的。但是它实际上就是一棵2-3树转换成对应的二叉查找树，就这么简单。

那为什么想把2-3树转换成别的树呢？2-3树不好吗？

因为用不同的数据类型表示2-结点和3-结点实现大多数操作并不方便，因为需要处理的情况实在太多。我们需要维护两种不同类型的结点，将结点从一种数据类型转换到另一种数据类型，等等。实现这些不仅需要大量代码，而且它们所产生的额外开销可能会使算法比标准的二叉查找树更慢。
平衡一棵树的初衷是为了消除最坏情况，但我们希望这种保障所需的代码能够越少越好，而在此之前我们已经完美地实现了二叉查找树，所以就在思考能不能将2-3树做一点小改变转换成二叉查找树，就能运用二叉查找树现成的代码了。

为什么isRed()不是Node类的方法呢？那样node.isRed()直接返回当前node的颜色？
因为这样的话，x==null就无法返回颜色了！node为空的时候根本无法调用其isRed()啊！

实现

插入put()

  1. 如果当前子树的根结点为null，则创建node，它的颜色是RED（新插入结点的color均为红色）
  2. 如果当前子树的根结点不为空，比较即将插入的键key和根结点的key，如果小于根结点则递归插入左子树中，如果大于根结点则递归插入右子树中，如果相等，则找到，直接修改根结点的val
  3. 插入完成后，按照上述三个操作的规则进行检查，做对应的操作更改node颜色

红黑树的性质

所有基于红黑树的符号表实现都能保证操作的运行时间为对数级别。

性能分析

  • 无论键的插入顺序如何，红黑树都几乎是完美平衡的。一棵大小为N的红黑树的高度不会超过$2lgN$。
  • 一棵大小为N的红黑树，根结点到任意结点的平均路径长度为～$1.00lgN$

API

红黑树最吸引人的一点是它的实现中最复杂的代码仅限于put()（和删除）方法。
二叉查找树中查找最大和最小键/select/rabk/floor/ceiling/范围查找方法不需要做任何修改即可继续使用，因为这些操作不涉及到结点的颜色。
