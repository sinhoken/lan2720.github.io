基于有序数组的二分查找

发表于 2015-10-24   |  

algs4中讲到了用一对平行的数组来实现符号表（symbol table），就需要实现put()和get()两个方法，两者都涉及到对某一给定的key在有序数组中的查找，比较高效的方法即二分查找。

BinarySearchST类的成员变量和构造函数

在类的定义处BinarySearchST<Key extends Comparable<Key>, Value>即使用了泛型，并且注明了Key实现了Comparable接口。
同样，可以看到后面的keys定义中new出的是一个Comparable类型的对象组成的数组，后来转成Key[]；vals new出的是Object类型的对象数组，因为只有键进行比较（方便进行排序），value不需要有比较方法。keys和vals是两个关键的平行数组。

rank()方法

这份实现的核心是rank()方法，它返回表中小于给定键的键的数量，也就是说如果键key存在当前的符号表中，它在有序数组中的idx；如果键key不在当前的符号表中，返回它应该插入位置的idx。
话不多说，上代码。

  1. 首先，定义两个指针，lo指向数组开头元素，hi指向数组末尾元素。
  2. 只有lo>hi时才跳出循环，跳出循环也就是当前的keys数组中没有找到指定的key，那么就返回key应该插入的idx，也就是lo。为什么是lo稍后再解释。
  3. 进入循环中，找到中间索引mid，我们要找的目标Key对象key，和keys[mid]这个Key对象进行比较，根据比较的大小，确定下一步的查找方向。为什么能比较大小？因为Key类实现了Comparable接口，所以调用compareTo()方法即可。
  4. 如果key小于keys[mid]，也就是key应该排在mid前，所以hi=mid-1，圈定前半个数组元素。
  5. 如果key大于keys[mid]，同上，应该往mid后面找lo=mid+1。
  6. 如果key等于keysp[mid]，说明正好找到，直接返回mid。

好，至此就只剩一个问题了，如果keys中不存在key这个键应该返回什么idx呢？

  • lo=hi=mid之后，cmp<0，hi=mid-1，如下图所示


这时，我们的目标key=6，不在keys中，但是要插入的话，应该在5之后，也就是现在7的idx，而现在指针lo指向7，因此return lo即可

  • lo=hi=mid之后，cmp>0，lo=mid+1，如下图所示

get()方法

分析完rank()之后，再看get()方法就没任何障碍了。

  1. 如果本身符号表中就不存在任何键值对，那么肯定找不到目标key，不浪费时间直接返回null
  2. 否则，就用rank()去二分查找目标key。要么找到，返回key现在的idx；要么找不到，但是返回应该插入的idx
  3. if (i < N && keys[i].compareTo(key) == 0)这句就是检测返回的idx对应的key究竟是不是目标key，因为没找到我们也会返回一个应该插入的idx位置，但是keys中现在这个idx上的键并不是我们要找的key
  4. 如果确实对应上了目标键，就返回它在vals数组中对应的值vals[i]
  5. 如果对应不上，说明目标key不存在符号表中，仍返回null

put()

  1. 如果设置了val=null，也就是用户自行想删除这个key，那么调用delete(key)方法
  2. 如果val不为空，就看这个key是否已经存在keys中。如果存在，返回idx；如果不存在，返回应该插入的idx，这正是rank()完成的工作
  3. 接下来就是两种情况：如果key已经存在，直接修改vals[i] = val即可；如果key不存在，就需要将当前的键值对插入到合适的i位置
  4. 因为要在idx=i插入新元素，因此当前的i~(N-1)的全部键值对都要向后移动一个空格以容纳新对象
  5. for循环完成了移动之后，key和val就可以赋值到现在的keys[i]和vals[i]了
