min-char-rnn代码解读

发表于 2015-08-09   |  

今天读了大神Andrej Karpathy关于rnn的博客，对于simple rnn（也叫Elman neuron network）有了了解，算是入门了~

Karpathy在博客中给出了一段python/numpy实现的char-rnn代码。本篇就针对这段代码进行一个解读。

首先，导入数据。文本数据存储input.txt文件中，而且都是连接在一起中间没有换行符等多余字符的。
导入后，data是str类型变量。set(data)之后去除了data str中的重复字母。chars变量作为一个列表存储整个input文本中所有出现过的字母。
data_size是input str一共包含多少个字符，vocab_size是其中有多少种字符（重复的只记一次，无非就是大小写的26个字母加一些数字啊标点符号啊等等）

对于去重的字符列表chars，进行一个char-index的转换。char_to_ix[char_x]输入char_x就能返回对应的index；ix_to_char同理。

定义一些超参数。所谓超参数就是定义模型必须的一些参数，比如模型的层数，每个隐层的神经元数，输入层的神经元个数，学习率，正则化的惩罚参数等等。相对而言的参数，就是我们训练过程中需要学习的w和b了。

这段代码是对模型的参数w和b进行一个初始化。
参数如下图所示

Wxh
: 输入层和隐层之间的w，(n_hid， n_vocab)

Whh
: context layer到隐层之间的w，(n_hid， n_hid)

Why
: 隐层到输出层之间的w，(n_vocab，n_hid )

bh
: 隐层的偏置（加在隐层神经元上$w\cdot x_{in} + bh$）

by
: 输出层的偏置（加在输出层上$w\cdot a_{hid} + by$）

loss function

loss function传入了三个参数，输入，目标和预测值。为什么是这三个参数。因为传入inputs，经过feedforward得到yprev才能和targets进行比较算出loss function的值。所以这三个参数缺一不可。
【注意】这里的inputs和targets都是int组成的list。每一个int都表示在chars列表中的index，而不是直接输入char。

【这几个字典是存储每一时刻的每一层输出状态的，它们的key都是t】
接下来是前向传播过程。
inputs有多长就有多少个step，每一个step也相当于t。
当t = 0时，xs[0] = 一个全零vector（维度是vocab_size，也就是和chars列表同型），但是这个vector在t时刻的input位置上赋值为1，表示xs把当前输入的字符用one-hot表示了。
将这个one-hot表示的xs[t] vector（代表当前的输入）和上一时刻的隐层状态hs[t-1]还有bh传输给隐层，hs[t] = np.tanh(np.dot(Wxh, xs[t]) + np.dot(Whh, hs[t - 1]) + bh)就计算出了当前时刻的hs[t]。先看看tanh()内的两个matrix是否可以相加。
Wxh是(n_hid，n_vocab)，xs[t]是(n_vocab，1)，点乘后得到(n_hid，1)矩阵；Whh是(n_hid，n_hid)，hs[t-1]是(n_hid，1)，点乘后得到(n_hid，1)，因此这两个矩阵同型，可以相加。得到的hs[t]是(n_hid，1)型vector。
隐层还要向前传播给输出层：ys[t] = np.dot(Why, hs[t]) + by，Why是(n_vocab，n_hid)，点乘后得到(n_vocab，1)，也就得到了当前时刻t的输出ys[t]。

先计算出ps[t]，实际上这表示输出层是一个softmax层，ps[t]是softmax层的输出。
ps[t]只进行了element-wise的计算，所以和ys[t]同型(n_vocab，1)，每一个位置对应chars中一个字符，值为下一时刻该字符出现的概率，这样就计算出每个字符（chars列表中的）在下一时刻出现的概率。
而targets向量在t时刻有一个输出目标字符（是用字符的index表示的），在ps[t]向量中取出这个目标值在下一时刻出现的概率，将这个概率的NLL累加到loss，t时刻的前向传播就完成了。接着回到输出层，t+1时刻重复上述过程，loss值累加t+1时刻目标字符出现概率的NLL，不断重复。
