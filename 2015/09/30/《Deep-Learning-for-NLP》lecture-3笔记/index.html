Softmax函数

之前的blog中也讲过softmax，但是到底什么是softmax，它衡量的是什么东西呢？
softmax函数其实和logistic regression中的sigmoid函数一样，它属于激活函数，但是它只能用在多分类任务中做激活函数。
它的功能是接受一个输入x，输出其属于每一分类的概率，由于它输出的每一个分类都是概率，所以值在(0,1)之间，且总和为1。这个输出的分类由一个概率计算公式决定，如下：

目标函数（损失函数）：Negative Log Likelihood

softmax仅仅是激活函数，输出的类别概率越大越好，但是由于我们需要进行梯度下降，所以要转变成最小化问题，就有了negative log likelihood，听起来高大上实际就是对概率取了个$-logP$而已。

推广：cross entropy

之前说交叉熵只能用于二分类问题，其实正相反，它和NLL在多分类上的推广：

$q(c)$是每一个类别的概率vector，上面计算过。
单分类时：
$p(c)$是one-hot label：[0, 0, …, 1, 0, 0, …, 0]，只有正确的类别是1，其他为0
显然$H(p,q)$就简化成了NLL
多分类时：
$p(c)$不仅仅属于一个类别，因此不是one-hot，$H(p,q)$也就计算了多个分类的概率之和

cross entropy实际上是NLL在多类别分类中的推广

这里的“多类别”指的是每一个instance不止属于一个class，可以同属于多个class的分类。

word vector的问题

比如一个简单的word classification问题。
有两种选择：

  • 在训练过程中不更新word vectors，只训练softmax的$W$矩阵
  • 既更新$W$，也根据特定任务更新word vectors

那么第二种方法有什么优缺点呢？
pro：更好的适应了训练数据
con：使得泛化能力变差。因为在vector space中words根据specified-task进行了移动，softmax相应的predict出的类别也会有变化，那些没有在当前任务中进行update的那些words就被分到错误的类别。
